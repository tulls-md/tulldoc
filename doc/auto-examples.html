Автопримеры

Tulldoc автоматически создаёт секцию примеров для каждого пропа компонента, тип которого - union of literals. Для работы нужен componentsDir.

Как это работает

  1. Tulldoc находит исходный файл компонента в componentsDir
  2. Читает TypeScript-тип первого параметра функции
  3. Для каждого пропа с enumerable-типом ("a" | "b" | "c") создаёт секцию
  4. Рендерит компонент для каждого значения, передавая defaultArgs + текущее значение

DocAutoExample

Запись в examples с полем prop настраивает автопример конкретного пропа:

examples: [
  {
    prop: "variant",
    title: "Варианты",
    description: "Визуальные стили кнопки.",
    view: "row",
    defaultArgs: { children: "Кнопка" },
  },
];

Поля

ПолеТипПо умолчаниюОписание
propkeyof Props-Проп, значения которого перебираются (обязательное)
titlestringимя пропаЗаголовок секции
descriptionstringОписание под заголовком
viewExampleView"row"Раскладка вариантов
defaultArgsPartial<Props>Переопределяет defaultArgs документа для этого примера
disabledbooleanfalsetrue - убрать автопример этого пропа

Порядок примеров

Порядок в секции «Примеры» определяется так:

  1. Записи из examples[] - в том порядке, в котором они написаны
  2. Пропсы, не упомянутые в examples[], - в порядке объявления в TypeScript-типе

Если проп упомянут в examples[] с disabled: true, он полностью пропускается.

Раскладки ExampleView

"row" (по умолчанию)

Все варианты в одном превью, горизонтально. Для небольших компонентов.

"grid"

Все варианты в одном превью, сеткой. Для одинаково крупных элементов.

"cell"

Каждый вариант в отдельном превью с подписью. Для крупных или независимых компонентов.

Поддерживаемые типы пропсов

ТипРезультат
"a" | "b" | "c"Три варианта: a, b, c
booleanДва варианта: true, false
1 | 2 | 3Три варианта: 1, 2, 3
stringОшибка - тип не enumerable
numberОшибка - тип не enumerable
ReactNodeОшибка - тип не enumerable

Если для пропа нельзя перечислить значения, tulldoc выводит сообщение об ошибке прямо в секции примера (в блоке DocNotice). Этот проп не блокирует остальные.

Отключение автопримера

Если автопример для конкретного пропа не нужен:

examples: [
  { prop: "disabled", disabled: true },
  { prop: "loading", disabled: true },
];

Пример полного использования

export default function (): DocMeta<typeof Button> {
  return {
    component: Button,
    defaultArgs: { children: "Кнопка" },
    examples: [
      {
        prop: "variant",
        title: "Стиль",
        view: "row",
      },
      {
        prop: "size",
        title: "Размер",
        defaultArgs: { children: "Текст кнопки" }, // переопределяет defaultArgs документа
      },
      { prop: "disabled", disabled: true }, // убираем автопример для disabled
    ],
  };
}
