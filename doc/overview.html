Автодокументирование компонентов

Этот раздел требует аддон @tulls-md/tulldoc-code - устанавливается отдельно (npm install @tulls-md/tulldoc-code) и подключается плагином componentDocs.

Файл .doc.tsx - декларативное описание компонента. Вместо того чтобы писать статью вручную, вы описываете: какой компонент документировать, какие примеры показать, какие пропсы использовать по умолчанию. Tulldoc строит страницу сам.

Когда использовать .doc.tsx, а когда .mdx

.doc.tsx.mdx
Документация компонента UI-библиотекиОбзорные статьи, гайды, changelog
Нужна таблица API и автопримерыПроизвольная структура страницы
Примеры привязаны к пропсам компонентаИллюстрации не зависят от пропсов

Один slug не может иметь одновременно .mdx и .doc.tsx - tulldoc выдаст ошибку.

Минимальный пример

// src/content/components/button.doc.tsx
import { Button } from "@my-org/ui";
import type { DocMeta } from "@tulls-md/tulldoc-code";

export default function (): DocMeta<typeof Button> {
  return {
    component: Button,
    description: "Кнопка для выполнения действий.",
  };
}

Из этого файла tulldoc автоматически генерирует страницу с:

  • Заголовком (имя функции компонента)
  • Описанием
  • Автопримерами для каждого пропа (если тип - union of literals)
  • Секцией API с таблицей всех пропсов

Что генерируется автоматически

Заголовок и TOC

h1 - из title или имени компонента. Все секции (анатомия, примеры, API) получают уникальные id и попадают в оглавление страницы.

Автопримеры

Для каждого пропа с типом union of literals создаётся секция с вариантами. Достаточно указать defaultArgs, чтобы обязательные пропсы были заполнены.

Таблица API

Пропсы компонента извлекаются из TypeScript - название, тип, обязательность, значение по умолчанию (из деструктуризации) и JSDoc-описание.

Подключение плагина componentDocs

Для работы .doc.tsx-файлов добавьте плагин componentDocs из аддона в createDocSource:

import { createDocSource } from "@tulls-md/tulldoc/server";
import { componentDocs } from "@tulls-md/tulldoc-code/server";

createDocSource({
  contentDir: join(process.cwd(), "src/content"),
  importContent: (path) => import(`./content/${path}.mdx`),
  plugins: [
    componentDocs({
      importDoc: (path) => import(`./content/${path}.doc.tsx`), // <- обязателен
      componentsDir: join(process.cwd(), "../../packages/ui/src/components"), // <- для API
      examplesDir: join(process.cwd(), "src/examples"), // <- для примеров
    }),
  ],
  lang: "ru",
});
  • importDoc - обязателен, импортирует .doc.tsx-документы
  • componentsDir - нужен для таблицы API и автопримеров
  • examplesDir - нужен для mainExample и ручных примеров
