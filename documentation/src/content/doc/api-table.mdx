# Таблица API

Секция **API** генерируется автоматически для каждого `.doc.tsx`-документа.
Tulldoc анализирует TypeScript-тип первого параметра компонента через compiler API
и строит таблицу пропсов без какого-либо ручного описания.

## Что показывается

| Колонка      | Источник                              |
| ------------ | ------------------------------------- |
| Имя          | Имя свойства TypeScript-типа          |
| Тип          | Текст типа (нормализованный)          |
| Обязательный | Свойство без `?`                      |
| По умолчанию | Значение в деструктуризации параметра |
| Описание     | JSDoc-комментарий над свойством       |

## JSDoc-описания

Добавляйте комментарии прямо над свойствами интерфейса:

```ts
interface ButtonProps {
  /** Визуальный стиль. Влияет на цвет и обводку. */
  variant?: "primary" | "secondary" | "ghost";

  /**
   * Блокирует взаимодействие и применяет стиль недоступности.
   * Форма продолжает принимать значение поля.
   */
  disabled?: boolean;

  children: React.ReactNode;
}
```

Многострочные JSDoc объединяются в одну строку.

## Значения по умолчанию

Tulldoc читает значения по умолчанию из деструктуризации параметра функции:

```ts
export function Button({
  variant = "primary",
  size = "md",
  disabled = false,
  children,
}: ButtonProps) {
  /* ... */
}
```

В таблице появятся: `variant` -> `"primary"`, `size` -> `"md"`, `disabled` -> `false`.
Значение для `children` не показывается - его нет в деструктуризации.

## Нормализация типов

Типы перед отображением проходят нормализацию:

- `React.ReactNode` -> `ReactNode` (убирается префикс `React.`)
- Однослойный не-экспортируемый алиас разворачивается: `type Size = "sm" | "md"` -> `"sm" | "md"`
- Типы из `node_modules` остаются по имени алиаса
- Лишние пробелы нормализуются

```ts
// ButtonProps.tsx
type InternalSize = "sm" | "md" | "lg"; // не экспортируется -> разворачивается

export interface ButtonProps {
  size?: InternalSize; // покажет: "sm" | "md" | "lg"
}
```

## Порядок пропсов

1. **Собственные пропсы** - объявленные в том же файле, что и компонент
2. **Унаследованные пропсы** - из других файлов (например, из `HTMLButtonElement`)

Это позволяет собственным пропсам всегда оказываться вверху таблицы.

## Пустая таблица

Если компонент не принимает пропсов (или параметр не найден), таблица заменяется текстом:

> Компонент не принимает пропсов.

## Требования к componentsDir

Таблица API работает только при наличии `componentsDir` в `createDocSource`.
Без него tulldoc не может найти файл компонента для анализа.

```ts
createDocSource({
  componentsDir: join(process.cwd(), "../../packages/ui/src/components"),
  // ...
});
```

Tulldoc ищет файл компонента в `componentsDir`, опираясь на имя функции (или `componentName`).
