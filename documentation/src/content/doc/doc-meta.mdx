# Интерфейс DocMeta

`DocMeta<C>` - это объект, который возвращает функция `.doc.tsx`. Тип импортируется из
аддона `@tulls-md/tulldoc-code` (`import type { DocMeta } from "@tulls-md/tulldoc-code"`).
Параметр `C` - тип документируемого компонента; он обеспечивает типизацию `defaultArgs`
и `examples`.

```ts
interface DocMeta<C extends ComponentType<any>> {
  component: C;
  title?: string;
  description?: string;
  componentName?: string;
  mainExample?: ReactElement | Partial<ComponentProps<C>>;
  mainExampleView?: ExampleView;
  defaultArgs?: Partial<ComponentProps<C>>;
  examples?: DocExample<ComponentProps<C>>[];
  subcomponents?: DocSubcomponent[];
  anatomy?: ReactNode;
}
```

## Поля

### component (обязательное)

Сам компонент. Tulldoc использует его для рендеринга примеров и нахождения
исходного файла (через `componentsDir`):

```ts
import { Button } from "@my-org/ui";

return { component: Button };
```

### title

Заголовок страницы. По умолчанию - имя функции компонента, извлечённое из AST.

```ts
return { component: Button, title: "Кнопка" };
```

### description

Описание под заголовком. Поддерживает markdown: `` `код` ``, **жирный**,
_курсив_, ссылки и списки. Упоминание пропа в инлайн-коде становится ссылкой
на его строку в таблице API.

### componentName

Явное имя компонента для отображения в коде примеров и поиска исходного файла.

Нужен в трёх случаях:

1. **Минификация** - имя функции не сохранилось в бандле
2. **Обёртки** - компонент экспортируется через `forwardRef`, `memo` или HOC
3. **Составные компоненты** - JSX-имя содержит точку: `Layout.Header`

```ts
// вместо имени функции forwardRef
return { component: Button, componentName: "Button" };

// составной компонент: Layout.Header -> ищет функцию LayoutHeader
return { component: Layout.Header, componentName: "Layout.Header" };
```

При поиске файла точки игнорируются: `"Layout.Header"` -> ищет функцию `LayoutHeader`.

### defaultArgs

Базовые пропсы, передаваемые компоненту во всех автопримерах. Используйте их
для обязательных пропсов или наполнения по умолчанию:

```ts
return {
  component: Button,
  defaultArgs: { children: "Кнопка" },
};
```

Конкретный автопример может переопределить `defaultArgs` через своё поле `defaultArgs`.
Значения мерджатся: `{ ...docDefaultArgs, ...exampleDefaultArgs }`.

### examples

Список настроек примеров - автоматических и ручных. Подробнее в разделах
«Автопримеры» и «Ручные примеры».

### mainExample

Главный пример, показываемый сразу под описанием, перед анатомией:
готовый компонент-пример (`ReactElement`) или объект пропсов, из которого
пример и код-блок генерируются автоматически.
Подробнее в разделе «Главный пример».

### mainExampleView

Вид главного примера: `"plain"` отключает точечный канвас.

### subcomponents

Подкомпоненты сборного компонента - для каждого в секции API рендерится
отдельная таблица пропсов:

```ts
interface DocSubcomponent {
  component: ComponentType<any>;
  componentName?: string; // как у DocMeta.componentName
  description?: string; // markdown
}
```

Подробнее в разделе «Таблица API».

### anatomy

JSX-схема анатомии компонента. Подробнее в разделе «Анатомия».
