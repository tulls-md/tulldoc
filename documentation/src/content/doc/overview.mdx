# Автодокументирование компонентов

Файл `.doc.tsx` - декларативное описание компонента. Вместо того чтобы писать
статью вручную, вы описываете: какой компонент документировать, какие примеры показать,
какие пропсы использовать по умолчанию. Tulldoc строит страницу сам.

## Когда использовать .doc.tsx, а когда .mdx

| `.doc.tsx`                             | `.mdx`                            |
| -------------------------------------- | --------------------------------- |
| Документация компонента UI-библиотеки  | Обзорные статьи, гайды, changelog |
| Нужна таблица API и автопримеры        | Произвольная структура страницы   |
| Примеры привязаны к пропсам компонента | Иллюстрации не зависят от пропсов |

Один slug не может иметь одновременно `.mdx` и `.doc.tsx` - tulldoc выдаст ошибку.

## Минимальный пример

```tsx
// src/content/components/button.doc.tsx
import { Button } from "@my-org/ui";
import type { DocMeta } from "@tulls-md/tulldoc";

export default function (): DocMeta<typeof Button> {
  return {
    component: Button,
    description: "Кнопка для выполнения действий.",
  };
}
```

Из этого файла tulldoc автоматически генерирует страницу с:

- Заголовком (имя функции компонента)
- Описанием
- Автопримерами для каждого пропа (если тип - union of literals)
- Секцией **API** с таблицей всех пропсов

## Что генерируется автоматически

### Заголовок и TOC

`h1` - из `title` или имени компонента. Все секции (анатомия, примеры, API) получают
уникальные `id` и попадают в оглавление страницы.

### Автопримеры

Для каждого пропа с типом `union of literals` создаётся секция с вариантами.
Достаточно указать `defaultArgs`, чтобы обязательные пропсы были заполнены.

### Таблица API

Пропсы компонента извлекаются из TypeScript - название, тип, обязательность,
значение по умолчанию (из деструктуризации) и JSDoc-описание.

## Необходимые опции createDocSource

Для работы `.doc.tsx`-файлов нужно передать в `createDocSource`:

```ts
createDocSource({
  contentDir: join(process.cwd(), "src/content"),
  importContent: (path) => import(`./content/${path}.mdx`),
  importDoc: (path) => import(`./content/${path}.doc.tsx`), // <- обязателен
  componentsDir: join(process.cwd(), "../../packages/ui/src/components"), // <- для API
  examplesDir: join(process.cwd(), "src/examples"), // <- для примеров
  lang: "ru",
});
```

- `importDoc` - обязателен, если в `contentDir` есть `.doc.tsx`
- `componentsDir` - нужен для таблицы API и автопримеров
- `examplesDir` - нужен для `mainExample` и ручных примеров
