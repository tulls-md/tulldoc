# Для кого?

Tulldoc подойдёт всем, кому нужен сайт документации с удобной навигацией
и возможностью писать контент в Markdown.

## Типичные пользователи

**Команда продукта.** Нужно руководство пользователя, onboarding, FAQ —
пишете `.mdx`-файлы, получаете полноценный сайт.

**Разработчик open-source.** Документация живёт рядом с кодом в монорепо,
деплоится как часть релиза. Не нужны сторонние хостинги.

**Технический писатель.** Markdown-синтаксис, структура из папок, никаких CMS
и баз данных — только файлы в репозитории.

**Команда с внутренними процессами.** Runbook-ы, архитектурные решения, регламенты —
закрытый сайт, который можно задеплоить в собственную инфраструктуру.

**Разработчик React-библиотеки.** Документация компонентов с автогенерацией таблицы API,
живыми примерами и привязкой к TypeScript-типам — через `.doc.tsx`.

## Технические требования

Для создания сайта документации на основе `.mdx`-файлов:

| Требование | Версия |
| ---------- | ------ |
| Next.js    | ≥ 16   |
| React      | ≥ 19   |
| Node.js    | ≥ 18   |

Документировать **можно что угодно**: текстовые гайды, API-справочники,
пользовательские сценарии — TypeScript и React в самом контенте не нужны.

React и TypeScript нужны только вам как разработчику, который настраивает сайт.
Это стандартные требования Next.js.

## Когда нужен TypeScript в документируемом проекте

**Только** если вы хотите использовать `.doc.tsx` для автодокументирования React-компонентов:
автопримеры из пропсов и таблица API, которая генерируется из TypeScript-типов.

Для всего остального TypeScript в документируемом проекте не нужен.

## Когда tulldoc не подойдёт

- **Нужен визуальный редактор** — tulldoc работает с файлами в репозитории, не через CMS-интерфейс
- **Нужен готовый дизайн «из коробки»** — tulldoc даёт структуру, стили нужно подключить самому
- **Vue / Svelte / Angular-компоненты с автодокументированием** — `.doc.tsx` поддерживает только React

## Примеры сценариев

**Документация API.** Эндпоинты, параметры запросов, примеры ответов — всё в `.mdx`-файлах
с подсветкой кода, сгруппированное по разделам через папки.

**Руководство пользователя продукта.** Пошаговые инструкции, скриншоты, советы —
документируйте хоть принципы работы микроволновки, структура не ограничивает.

**Дизайн-система.** Гайдлайны, принципы, компоненты с живыми примерами и таблицей пропсов.

**Монорепо.** Документация живёт в `packages/docs` рядом с исходниками,
`componentsDir` указывает напрямую на компоненты.
