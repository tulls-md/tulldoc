# Начало работы

## Установка

```bash
npm install @tulls-md/tulldoc
```

Peer-зависимости: `next >= 16`, `react >= 19`, `react-dom >= 19`.

Документирование React-компонентов из кода (таблицы пропсов, автопримеры, `.doc.tsx`) - отдельный аддон, устанавливается дополнительно:

```bash
npm install @tulls-md/tulldoc-code
```

## Структура файлов

Минимальный проект выглядит так:

```
my-docs/
├── next.config.ts
├── src/
│   ├── app/
│   │   ├── layout.tsx
│   │   ├── page.tsx
│   │   └── [...slug]/
│   │       └── page.tsx
│   ├── content/
│   │   └── overview.mdx
│   ├── docs.ts
│   └── mdx-components.tsx
```

## next.config.ts

Оберните конфиг через `withTulldoc` - он подключает MDX-плагин и нужные `pageExtensions`.

```ts
import { withTulldoc } from "@tulls-md/tulldoc/config";

export default withTulldoc();
```

Дополнительные опции `NextConfig` передаются первым аргументом:

```ts
export default withTulldoc({
  transpilePackages: ["@my-org/ui"],
});
```

## mdx-components.tsx

Next.js требует файл `mdx-components.tsx` в корне `src/` для глобальных MDX-компонентов.
Подключите компоненты tulldoc (`CodeBlock`, `Preview` и другие):

```tsx
import type { MDXComponents } from "mdx/types";
import { getMDXComponents } from "@tulls-md/tulldoc";

export function useMDXComponents(): MDXComponents {
  return getMDXComponents();
}
```

Чтобы добавить свои компоненты - передайте их в `getMDXComponents`:

```tsx
import { MyCallout } from "@/components/callout";

export function useMDXComponents(): MDXComponents {
  return getMDXComponents({ Callout: MyCallout });
}
```

## docs.ts

Создайте `src/docs.ts` - центральный источник данных для роутера.

```ts
import { join } from "path";
import { createDocSource } from "@tulls-md/tulldoc/server";

export const docs = createDocSource({
  contentDir: join(process.cwd(), "src/content"),
  importContent: (path) => import(`./content/${path}.mdx`),
  lang: "ru",
});
```

### Все опции `createDocSource`

| Опция           | Тип                                             | Описание                                             |
| --------------- | ----------------------------------------------- | ---------------------------------------------------- |
| `contentDir`    | `string`                                        | Абсолютный путь до папки с контентом                 |
| `importContent` | `(path) => Promise<{ default: ComponentType }>` | Динамический импорт `.mdx`-файлов                    |
| `plugins`       | `TulldocPlugin[]`                               | Плагины-расширения (например `componentDocs` аддона) |
| `lang`          | `string`                                        | Язык UI-строк (`"ru"` или `"en"`)                    |

`importContent` обязателен, если в `contentDir` есть `.mdx`-файлы.

Документирование компонентов (`.doc.tsx`, таблицы пропсов, автопримеры) подключается плагином `componentDocs` из аддона `@tulls-md/tulldoc-code` - см. раздел «Автодокументирование компонентов».

## App Router

Три файла в `app/` подключают роутер:

**`src/app/layout.tsx`** - лэйаут с боковой навигацией. Здесь же один раз
подключаются стили tulldoc (`@tulls-md/tulldoc/styles.css` - reset, тема,
шрифты):

```tsx
import "@tulls-md/tulldoc/styles.css";
import { docs } from "@/docs";

export default docs.Layout;
```

Если используете аддон `@tulls-md/tulldoc-code`, добавьте сюда же его стили:

```tsx
import "@tulls-md/tulldoc/styles.css";
import "@tulls-md/tulldoc-code/styles.css";
```

**`src/app/page.tsx`** - корневая страница, редиректит на первый раздел:

```tsx
import { docs } from "@/docs";

export default docs.IndexPage;
```

**`src/app/[...slug]/page.tsx`** - страница документа:

```tsx
import { docs } from "@/docs";

export const dynamicParams = false;
export const generateStaticParams = docs.generateStaticParams;
export const generateMetadata = docs.generateMetadata;

export default docs.Page;
```
