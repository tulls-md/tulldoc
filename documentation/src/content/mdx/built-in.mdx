# Встроенные компоненты

Функция `getMDXComponents` регистрирует несколько компонентов и переопределяет HTML-теги.
Всё это работает в каждом `.mdx`-файле без дополнительных импортов.

## CodeBlock

Все блоки кода в MDX автоматически рендерятся через `CodeBlock`.
Подсветка синтаксиса выполняется через [Shiki](https://shiki.style/) с темой `github-light`.

````mdx
```tsx
function Hello() {
  return <p>Hello</p>;
}
```
````

### Пропы

`CodeBlock` можно использовать и явно - если нужны дополнительные опции:

```mdx
<CodeBlock
  lang="bash"
  collapsible
  showLabel="Показать установку"
  hideLabel="Скрыть"
>
  npm install @my-org/ui
</CodeBlock>
```

| Проп          | Тип       | По умолчанию       | Описание                                         |
| ------------- | --------- | ------------------ | ------------------------------------------------ |
| `children`    | `string`  | -                  | Текст кода                                       |
| `lang`        | `string`  | `"text"`           | Язык для подсветки (`tsx`, `bash`, `json` и др.) |
| `collapsible` | `boolean` | `false`            | Скрывает код за кнопкой «Посмотреть код»         |
| `showLabel`   | `string`  | `"Посмотреть код"` | Текст кнопки раскрытия                           |
| `hideLabel`   | `string`  | `"Скрыть код"`     | Текст кнопки сворачивания                        |

Блоки кода внутри `ComponentPreview` и `ExampleVariants` всегда `collapsible: true`.

## Preview

Компонент `Preview` оборачивает содержимое в изолированный блок с нейтральным фоном.
Используйте его, когда хотите показать компонент прямо в тексте статьи.

```mdx
<Preview>
  <Button variant="primary">Нажми меня</Button>
</Preview>
```

### Пропы

| Проп       | Тип         | Описание                                |
| ---------- | ----------- | --------------------------------------- |
| `children` | `ReactNode` | Компонент для отображения               |
| `height`   | `number`    | Фиксированная высота области в пикселях |

Проп `height` нужен для компонентов, которые занимают всё окно браузера
(`position: fixed`, `height: 100dvh`). Без него превью схлопнется в нулевую высоту.

```mdx
<Preview height={300}>
  <Dialog open />
</Preview>
```

## Заголовки h2 и h3

Теги `h2` и `h3` автоматически получают атрибут `id`, сформированный из текста
заголовка через slugify. Это заполняет оглавление (Table of Contents) страницы.

```mdx
## Установка -> id="ustanovka"

### Через npm -> id="cherez-npm"
```

`h1` не трогается - он используется как заголовок страницы и в TOC не включается.
Заголовки `h4`-`h6` также не получают `id` и в TOC не отображаются.
