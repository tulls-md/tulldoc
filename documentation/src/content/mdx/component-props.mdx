# ComponentProps

`ComponentProps` рендерит таблицу пропсов компонента, извлечённых из TypeScript через
статический анализ. Ни вручную описывать пропсы, ни поддерживать их актуальность не нужно.

```mdx
<ComponentProps src="button/button" />
```

## Пропы

| Проп   | Тип      | По умолчанию              | Описание                                                         |
| ------ | -------- | ------------------------- | ---------------------------------------------------------------- |
| `src`  | `string` | -                         | Путь к файлу компонента относительно `componentsDir`, без `.tsx` |
| `name` | `string` | PascalCase от имени файла | Имя экспортируемого компонента                                   |

Если компонент называется так же, как файл в PascalCase, `name` не нужен:

```
componentsDir/
└── button/
    └── button.tsx          ->  src="button/button"
                               name автоматически = "Button"
```

Если имя отличается, укажите явно:

```mdx
<ComponentProps src="button/button" name="ButtonRoot" />
```

## Что показывается в таблице

Для каждого пропа первого параметра компонента извлекается:

| Колонка      | Источник                               |
| ------------ | -------------------------------------- |
| Имя          | имя свойства в TypeScript-типе         |
| Тип          | текст типа (нормализованный)           |
| Обязательный | отсутствие `?` в типе                  |
| По умолчанию | значение из деструктуризации параметра |
| Описание     | JSDoc-комментарий над свойством        |

### Пример

```tsx
interface ButtonProps {
  /** Визуальный стиль кнопки */
  variant?: "primary" | "secondary" | "ghost";
  /** Размер кнопки */
  size?: "sm" | "md" | "lg";
  /** Отключает взаимодействие */
  disabled?: boolean;
  children: React.ReactNode;
}

export function Button({
  variant = "primary",
  size = "md",
  disabled = false,
  children,
}: ButtonProps) {
  /* ... */
}
```

Таблица для этого компонента:

| Проп       | Тип                                   | Обязательный | По умолчанию | Описание                 |
| ---------- | ------------------------------------- | :----------: | ------------ | ------------------------ |
| `variant`  | `"primary" \| "secondary" \| "ghost"` |              | `"primary"`  | Визуальный стиль кнопки  |
| `size`     | `"sm" \| "md" \| "lg"`                |              | `"md"`       | Размер кнопки            |
| `disabled` | `boolean`                             |              | `false`      | Отключает взаимодействие |
| `children` | `ReactNode`                           |      \*      |              |                          |

## Порядок пропсов

Сначала идут **собственные пропсы** компонента (объявленные в том же файле),
затем **унаследованные** (из других файлов, например, из `HTMLButtonElement`).

## Нормализация типов

- Префикс `React.` убирается: `React.ReactNode` -> `ReactNode`
- Однослойные не-экспортируемые алиасы разворачиваются: `type Size = "sm" | "md"` -> `"sm" | "md"`
- Типы из `node_modules` остаются по имени алиаса
- Лишние пробелы нормализуются

## JSDoc-описания

Описания берутся из JSDoc-комментариев над каждым свойством интерфейса:

```ts
interface InputProps {
  /**
   * Делает поле недоступным для редактирования.
   * В отличие от disabled, значение всё равно отправляется с формой.
   */
  readOnly?: boolean;
}
```

Многострочные комментарии объединяются в одну строку с пробелом.
