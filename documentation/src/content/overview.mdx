# Обзор

`@tulls-md/tulldoc` — платформа для создания сайтов документации на базе Next.js.

Напишите `.mdx`-файлы, положите их в папку — tulldoc соберёт сайт с боковой навигацией,
оглавлением страниц и красивой вёрсткой. Никаких шаблонов, никакой лишней конфигурации.

## Что можно документировать

Инструмент не привязан к предметной области:

- **Продукты и сервисы** — руководства пользователя, FAQ, onboarding
- **API и SDK** — справочники эндпоинтов, примеры кода, changelog
- **Внутренние процессы** — runbook-ы, архитектурные решения, регламенты
- **Open-source проекты** — гайды по установке, документация модулей
- **React-библиотеки компонентов** — с автогенерацией таблицы пропсов и живыми примерами _(опционально)_

Если это можно написать текстом — это можно задокументировать в tulldoc.

## Как это работает

**Файл → страница.** Каждый `.mdx`-файл в `contentDir` автоматически становится
страницей по соответствующему URL-пути.

**Папка → раздел.** Структура папок превращается в боковую навигацию.
Порядок и заголовки разделов управляются через `meta.json`.

**MDX → разметка.** Пишите обычный Markdown с возможностью вставлять React-компоненты
прямо в текст — таблицы, предупреждения, интерактивные блоки.

## Что входит

| Возможность           | Описание                                                            |
| --------------------- | ------------------------------------------------------------------- |
| **Файловый роутинг**  | `.mdx`-файлы в `contentDir` становятся страницами автоматически     |
| **Боковая навигация** | Строится из структуры папок; порядок управляется через `meta.json`  |
| **Вкладки**           | Папки с `_`-префиксом создают документ с вкладками                  |
| **MDX-компоненты**    | `CodeBlock`, `Preview` и автоматические якоря заголовков            |
| **SEO**               | `title` и `description` из frontmatter попадают в мета-теги         |
| **Пагинация**         | Кнопки «Предыдущая / Следующая страница» генерируются автоматически |

## Для React-библиотек: `.doc.tsx`

Если документируете компоненты React, tulldoc предлагает дополнительный формат файлов — `.doc.tsx`.
Он позволяет описать компонент декларативно: примеры по пропсам генерируются автоматически
из TypeScript-типов, таблица API извлекается из исходников без ручной поддержки.

Это отдельный аддон `@tulls-md/tulldoc-code` — устанавливается дополнительно, тем, кому нужна
только MDX-документация, его зависимости не нужны. Обычные `.mdx`-файлы работают без React и TypeScript.

## Точки входа пакета

| Путь                       | Назначение                                              |
| -------------------------- | ------------------------------------------------------- |
| `@tulls-md/tulldoc`        | UI-блоки и MDX-утилиты                                  |
| `@tulls-md/tulldoc/server` | Серверные хелперы: `createDocSource`, `getNavItems`     |
| `@tulls-md/tulldoc/config` | `withTulldoc` — обёртка для `next.config.ts`            |
| `@tulls-md/tulldoc-code`   | Аддон: документирование компонентов (ставится отдельно) |
