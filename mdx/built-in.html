Встроенные компоненты

Функция getMDXComponents регистрирует несколько компонентов и переопределяет HTML-теги. Всё это работает в каждом .mdx-файле без дополнительных импортов.

CodeBlock

Все блоки кода в MDX автоматически рендерятся через CodeBlock. Подсветка синтаксиса выполняется через Shiki с темой github-light.

```tsx
function Hello() {
  return <p>Hello</p>;
}
```

Пропы

CodeBlock можно использовать и явно - если нужны дополнительные опции:

<CodeBlock
  lang="bash"
  collapsible
  showLabel="Показать установку"
  hideLabel="Скрыть"
>
  npm install @my-org/ui
</CodeBlock>
ПропТипПо умолчаниюОписание
childrenstring-Текст кода
langstring"text"Язык для подсветки (tsx, bash, json и др.)
collapsiblebooleanfalseСкрывает код за кнопкой «Посмотреть код»
showLabelstring"Посмотреть код"Текст кнопки раскрытия
hideLabelstring"Скрыть код"Текст кнопки сворачивания

Блоки кода внутри ComponentPreview и ExampleVariants всегда collapsible: true.

Preview

Компонент Preview оборачивает содержимое в изолированный блок с нейтральным фоном. Используйте его, когда хотите показать компонент прямо в тексте статьи.

<Preview>
  <Button variant="primary">Нажми меня</Button>
</Preview>

Пропы

ПропТипОписание
childrenReactNodeКомпонент для отображения
heightnumberФиксированная высота области в пикселях

Проп height нужен для компонентов, которые занимают всё окно браузера (position: fixed, height: 100dvh). Без него превью схлопнется в нулевую высоту.

<Preview height={300}>
  <Dialog open />
</Preview>

Заголовки h2 и h3

Теги h2 и h3 автоматически получают атрибут id, сформированный из текста заголовка через slugify. Это заполняет оглавление (Table of Contents) страницы.

## Установка -> id="ustanovka"

### Через npm -> id="cherez-npm"

h1 не трогается - он используется как заголовок страницы и в TOC не включается. Заголовки h4-h6 также не получают id и в TOC не отображаются.
