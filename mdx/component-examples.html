ComponentExamples

ComponentExamples показывает все значения одного пропа как набор вариантов. Для каждого значения рендерится компонент и генерируется соответствующий JSX-код.

<ComponentExamples
  path="button/button-variants"
  prop="variant"
  view="row"
  defaultArgs={{ children: "Кнопка" }}
/>

Пропы

ПропТипПо умолчаниюОписание
pathstring-Путь к файлу примера относительно examplesDir, без .tsx
propstring-Имя пропа, значения которого перебираются
view"row" | "grid" | "cell""row"Раскладка вариантов
defaultArgsRecord<string, unknown>{}Дополнительные пропсы для каждого варианта

Структура файла примера

Файл должен иметь default export - компонент с параметром-объектом (без деструктуризации) и спредом {...props} в JSX:

// src/examples/button/button-variants.tsx
import { Button, type ButtonProps } from "@my-org/ui";

export default function ButtonVariantsExample(props: ButtonProps) {
  return <Button {...props} />;
}

Tulldoc подставит в {...props} конкретное значение для каждого варианта.

Требования к файлу

  • Default export - функция с одним параметром (не деструктурированным)
  • В JSX должен быть спред {...paramName}
  • Нет вложенных примеров из examplesDir (в отличие от ComponentPreview)

Как извлекаются значения пропа

Значения считываются из TypeScript-типа через статический анализ. Поддерживаются пропы с union of literals:

type ButtonVariant = "primary" | "secondary" | "ghost";

Если проп имеет тип string или number (не union), tulldoc выдаст ошибку - такой проп не enumerable и перебрать его значения невозможно. В этом случае используйте ComponentPreview с отдельными примерами.

Раскладки view

"row" (по умолчанию)

Все варианты в одной области превью, расположены горизонтально. Подходит для кнопок, тегов, бейджей.

"grid"

Все варианты в одной области превью, расположены сеткой. Подходит для карточек и элементов одинакового размера.

"cell"

Каждый вариант в отдельной области превью с подписью. Подходит для крупных компонентов или когда варианты нельзя смешивать в одном контексте.

defaultArgs

defaultArgs передаётся каждому варианту дополнительно к перебираемому пропу. Используйте его, чтобы задать обязательные пропсы или наполнение:

<ComponentExamples
  path="button/button-sizes"
  prop="size"
  defaultArgs={{ variant: "primary", children: "Кнопка" }}
/>
