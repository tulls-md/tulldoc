ComponentProps

ComponentProps рендерит таблицу пропсов компонента, извлечённых из TypeScript через статический анализ. Ни вручную описывать пропсы, ни поддерживать их актуальность не нужно.

<ComponentProps src="button/button" />

Пропы

ПропТипПо умолчаниюОписание
srcstring-Путь к файлу компонента относительно componentsDir, без .tsx
namestringPascalCase от имени файлаИмя экспортируемого компонента

Если компонент называется так же, как файл в PascalCase, name не нужен:

componentsDir/
└── button/
    └── button.tsx          ->  src="button/button"
                               name автоматически = "Button"

Если имя отличается, укажите явно:

<ComponentProps src="button/button" name="ButtonRoot" />

Что показывается в таблице

Для каждого пропа первого параметра компонента извлекается:

КолонкаИсточник
Имяимя свойства в TypeScript-типе
Типтекст типа (нормализованный)
Обязательныйотсутствие ? в типе
По умолчаниюзначение из деструктуризации параметра
ОписаниеJSDoc-комментарий над свойством

Пример

interface ButtonProps {
  /** Визуальный стиль кнопки */
  variant?: "primary" | "secondary" | "ghost";
  /** Размер кнопки */
  size?: "sm" | "md" | "lg";
  /** Отключает взаимодействие */
  disabled?: boolean;
  children: React.ReactNode;
}

export function Button({
  variant = "primary",
  size = "md",
  disabled = false,
  children,
}: ButtonProps) {
  /* ... */
}

Порядок пропсов

Сначала идут собственные пропсы компонента (объявленные в том же файле), затем унаследованные (из других файлов, например, из HTMLButtonElement).

Нормализация типов

  • Префикс React. убирается: React.ReactNode -> ReactNode
  • Однослойные не-экспортируемые алиасы разворачиваются: type Size = "sm" | "md" -> "sm" | "md"
  • Типы из node_modules остаются по имени алиаса
  • Лишние пробелы нормализуются

JSDoc-описания

Описания берутся из JSDoc-комментариев над каждым свойством интерфейса:

interface InputProps {
  /**
   * Делает поле недоступным для редактирования.
   * В отличие от disabled, значение всё равно отправляется с формой.
   */
  readOnly?: boolean;
}

Многострочные комментарии объединяются в одну строку с пробелом.
