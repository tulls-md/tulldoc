Файлы и навигация

Tulldoc строит навигацию напрямую из файловой системы. Каждый .mdx-файл в contentDir становится страницей, каждая папка — группой или табовым документом.

Содержимое файлов не ограничено: гайды, справочники, руководства пользователя, описания API — любой текст в Markdown.

MDX-файл как страница

Файл src/content/getting-started.mdx автоматически доступен по адресу /getting-started. Путь в URL - это путь к файлу без расширения, с разделителями /.

Каждый .mdx поддерживает YAML-frontmatter в начале файла:

---
title: Начало работы
description: Установка и минимальная настройка
---

# Начало работы

Контент страницы...

Поддерживаемые поля:

ПолеИспользование
titleМетатег <title>, заголовок в сайдбаре, пагинация
descriptionМетатег <meta name="description">

Если title не указан, заголовок в сайдбаре формируется из имени файла: my-page -> My Page.

Структура папок и группы

Папка без _-префикса создаёт группу в боковой навигации:

src/content/
├── overview.mdx                -> /overview
├── getting-started.mdx        -> /getting-started
└── components/                 <- группа
    ├── button.mdx             -> /components/button
    └── input.mdx              -> /components/input

Название группы в сайдбаре берётся из meta.json этой папки (title). Если meta.json нет - из имени папки в PascalCase: my-group -> My Group.

Группы могут быть вложенными - создавайте подпапки произвольной глубины.

meta.json

Файл meta.json в любой папке управляет заголовком группы и порядком элементов:

{
  "title": "Компоненты",
  "pages": ["button", "input", "select", "checkbox"]
}
  • title - название группы (или раздела) в сайдбаре
  • pages - список имён файлов/папок без расширений; задаёт порядок в сайдбаре

Элементы, не попавшие в pages, добавляются в конце в алфавитном порядке.

Внешние ссылки

Вместо имени файла в pages и header можно указать объект с label и href - получится внешняя ссылка:

{
  "pages": [
    "overview",
    { "label": "GitHub", "href": "https://github.com/acme/ui" }
  ],
  "header": [
    "guides",
    { "label": "npm", "href": "https://www.npmjs.com/package/acme-ui" }
  ]
}

Позиция объекта в массиве задаёт место ссылки в навигации. Работает на любом уровне: в корне сайдбара, внутри групп и в выпадающих меню шапки (через pages в meta.json папки-дропдауна). Внешняя ссылка помечается иконкой ↗, открывается в новой вкладке и не участвует в пагинации «вперёд/назад».

Ограничение: внутри табовых _-папок внешние ссылки не поддерживаются - вкладки описывают контент одного документа.

Табовые документы

Если страница должна иметь несколько вкладок (например, «Обзор» и «API»), создайте папку с префиксом _:

src/content/
└── _button/                    <- табовая папка
    ├── meta.json
    ├── overview.mdx           -> /button/overview
    └── api.mdx                -> /button/api

В URL префикс _ не входит: папка _button -> slug button. При переходе на /button открывается первая вкладка.

Порядок и подписи вкладок управляются через _button/meta.json:

{
  "pages": ["overview", "api"]
}

Название вкладки берётся из title во frontmatter каждого файла; если frontmatter нет - из имени файла в PascalCase.

Ограничение: внутри _-папки не может быть подпапок.

Пример полной структуры

src/content/
├── meta.json                        { "pages": ["overview", "guide", "reference"] }
├── overview.mdx
├── guide/
│   ├── meta.json                    { "title": "Руководство" }
│   ├── installation.mdx
│   └── configuration.mdx
└── reference/
    ├── meta.json                    { "title": "Справочник", "pages": ["api", "cli"] }
    ├── _api/
    │   ├── meta.json                { "pages": ["overview", "endpoints"] }
    │   ├── overview.mdx
    │   └── endpoints.mdx
    └── cli.mdx
