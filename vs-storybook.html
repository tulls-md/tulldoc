Tulldoc vs Storybook

Это сравнение актуально только если вы документируете React-компоненты. Для любого другого контента Storybook не рассматривается как альтернатива.

Главное отличие

Storybook — среда разработки компонентов. Вы пишете stories в процессе разработки, документация — побочный продукт.

Tulldoc — генератор сайта документации. Готовые компоненты описываются через .doc.tsx, примеры привязаны к TypeScript-типам.

Когда выбрать Storybook

  • Нужен hot reload и интерактивная среда при разработке компонента
  • Нужны Controls — интерактивно менять пропсы прямо в браузере
  • Нужно визуальное регрессионное тестирование (Chromatic)
  • Библиотека не на React или не на TypeScript

Когда выбрать Tulldoc

  • Нужен сайт документации, а не среда разработки
  • Важна автоматическая синхронизация таблицы API с TypeScript-типами
  • Документация включает статьи, гайды, changelog — не только компоненты
  • Документируемый проект не связан с React

Автопримеры vs Stories

В Storybook каждый вариант — это отдельная Story, написанная и поддерживаемая вручную.

В tulldoc автопримеры генерируются из TypeScript-типа: добавили значение в union — оно автоматически появится в документации.

// Storybook: нужна story для каждого варианта
export const Primary: Story = { args: { variant: "primary" } };
export const Secondary: Story = { args: { variant: "secondary" } };

// Tulldoc: достаточно одной записи
{ prop: "variant", view: "row" }

Совместное использование

Tulldoc и Storybook не исключают друг друга. Storybook — для разработки, tulldoc — для публичной документации. Стандартная схема: stories помогают в процессе разработки, tulldoc генерирует сайт для конечных пользователей библиотеки.
